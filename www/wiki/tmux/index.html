Using tmux

tmux is a terminal multiplexer — it lets you run persistent shell sessions that keep going after you close your SSH connection. On tharavad, it's essential for keeping servers alive.

Why you need tmux

When you SSH into tharavad and start a server, that server process is a child of your SSH session. When you close the terminal or lose connection, your SSH session ends — and everything it started gets killed automatically by the operating system.

tmux solves this by running a server of its own on the remote machine. You attach to it over SSH, run your stuff inside it, then detach (not exit) — the tmux server keeps running with all your processes inside, even after you disconnect.

Key concept: "Detach" means "step away but leave everything running." Closing your terminal without detaching first will kill your session and stop your server.

Creating a session

tmux new -s myproject
tmux
The tmux client. Used to create, attach to, and manage sessions.
new
Creates a new session. Short for new-session.
-s myproject
Names the session "myproject". Names make it easy to find and attach to specific sessions later. Use something descriptive — the project name, a port number, etc.

After running this, you're dropped into a new bash shell inside tmux. You'll notice a green status bar at the bottom of the terminal showing the session name and window number. That bar means you're inside tmux.

Running your server

Inside the tmux session, start your server normally:

# navigate to your project folder first
cd ~/myproject

# start your server
python3 -m http.server 45320

The server starts and logs appear in the terminal. Leave it running — don't press Ctrl+C.

Detaching (leaving without stopping)

To step away from the session while leaving everything running:

Press Ctrl + B, then release both, then press D.

This is the tmux prefix key sequence. Ctrl+B is the "attention" signal that tells tmux the next keystroke is a tmux command (not input to your shell). D means detach.

You'll see a message like [detached (from session myproject)] and you're back in your regular shell. Your server is still running inside tmux.

Do not press Ctrl+C to "exit" tmux — that kills the running process. Always detach with Ctrl+B D.

Re-attaching to a session

When you SSH back in later and want to check on your server:

tmux attach -t myproject
attach
Reconnects you to an existing session.
-t myproject
Specifies the target session by name. If you only have one session, you can just run tmux attach.

If you get "duplicate session" when creating — the session already exists. Just attach to it instead.

Listing sessions

tmux ls

Shows all running sessions, their names, how many windows they have, and when they were created. Use this to see what's running before attaching.

Killing a session

tmux kill-session -t myproject

Terminates the session and everything running inside it. Use this to fully stop a project (servers, processes, etc.).

Keyboard shortcuts reference

All tmux shortcuts start with the prefix Ctrl+B — press and release it, then press the key.

Shortcut What it does
Ctrl+B D Detach from session (leave it running)
Ctrl+B C Create a new window (tab) inside the session
Ctrl+B N Switch to the next window
Ctrl+B P Switch to the previous window
Ctrl+B 0–9 Switch to window by number
Ctrl+B % Split pane vertically (side by side)
Ctrl+B " Split pane horizontally (top/bottom)
Ctrl+B Arrow keys Move between panes
Ctrl+B S List and switch between sessions interactively
Ctrl+B ? Show all keybindings (press Q to exit)

Typical workflow for hosting

# 1. SSH in
ssh tharavad

# 2. Start a named tmux session
tmux new -s myapp

# 3. Inside tmux: navigate and start server
cd ~/myapp
python3 -m http.server 45320

# 4. Detach (Ctrl+B, then D)

# 5. Back in regular shell: expose subdomain
tinker caddy myapp 45320

# 6. Done — disconnect from SSH, server keeps running
