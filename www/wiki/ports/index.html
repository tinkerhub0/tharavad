Ports

A port is a numbered channel on a server that a process listens on for incoming connections. tharavad gives you a reserved port number that only your account can use, so your services don't collide with other users'.

What is a port?

Every process that accepts network connections does so on a specific port — a number from 1 to 65535. For example, web servers typically use port 80 (HTTP) or 443 (HTTPS). When you run a server on tharavad, you pick a port number and your server binds to it — meaning it listens for connections on that number.

Because many users share the same server, two processes cannot bind the same port at the same time. That's what tinker port solves: it reserves an available port number for your account so nobody else can grab it.

Info: You don't need to open firewall rules or configure anything else. As long as your server is running on the reserved port, it's accessible internally. Adding a subdomain via tinker caddy is what makes it public.

Reserving a port

tinker port
→ 45320
tinker
The tharavad CLI tool. Manages ports, subdomains, and routing for your account.
port
The subcommand. With no arguments, it reserves a new available port and prints the number.

The number printed (e.g. 45320) is now reserved for your account. Use this number when starting your server. You can reserve multiple ports — one per project.

Listing your reserved ports

tinker port list

Shows all ports currently reserved under your account. Useful if you've forgotten which port a project was using.

Releasing a port

tinker port release 45320
release
Frees the port so it can be used by others. Do this when you're done with a project.
45320
The port number to release. Replace with your actual port.
Auto-release: Ports are automatically released after 7 days of inactivity. If nothing has connected to the port in a week, it's freed. If your project stops being reachable after a week, re-run tinker port to get a new port, then restart your server and update your caddy subdomain.

Using a port in your server

Once reserved, pass your port number to whatever server you're running. Some examples:

Python static file server

# serves the current directory over HTTP
python3 -m http.server 45320
python3 -m http.server
Runs Python's built-in HTTP server module. Serves all files in the current directory as static files. No setup needed.
45320
The port to bind to. Replace with your reserved port number.

Node.js app

PORT=45320 node server.js

Or if your app reads from process.env.PORT, set it inline like above. Alternatively hardcode it in your app.

Go binary

./myapp --port 45320
