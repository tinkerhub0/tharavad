Environment

What's in your shell, how resources are allocated, and how your home directory is structured on tharavad.

Resource limits

Each user account runs in an isolated environment with the following resource limits enforced by the system:

ResourceLimit
Disk quota5 GB (your home directory)
RAM4 GB per user
CPU200% (equivalent to 2 cores)
Port reservationMultiple ports, auto-released after 7 days inactivity
Disk quota: Your 5 GB limit covers everything in your home directory — code, dependencies, databases, logs. If you install large node_modules folders or pip packages globally you'll eat into it fast. Check your usage with du -sh ~.

Shell

Your default shell is bash. You land in your home directory at /home/username when you SSH in. Standard bash config files apply — you can customise ~/.bashrc or ~/.bash_profile as you would on any Linux system.

# check your current shell
echo $SHELL

# check disk usage
du -sh ~

# check available memory
free -h

Pre-installed tools

The following tools are available in your shell without any installation:

python3
Python 3. Includes pip. Good for quick HTTP servers, scripts, and data work.
node / npm
Node.js runtime and npm package manager. Run JS apps and install packages.
go
Go compiler and toolchain. Build and run Go programs directly.
rust / cargo
Rust compiler and Cargo package manager. Build Rust projects.
git
Version control. Clone repos, commit, push, pull. SSH keys work for GitHub.
tmux
Terminal multiplexer. Keep servers running after disconnecting.
vim
Terminal text editor. Also available: nano if you prefer something simpler.
curl / wget
HTTP clients. Download files, hit APIs, test endpoints from the command line.
tinker
tharavad's own CLI. Manages port reservations and Caddy subdomains.

Home directory

Your home directory is at /home/username. It's private — other users cannot read it. Standard dot-files work as expected:

PathPurpose
~/Your home directory. All your files live here.
~/.bashrcBash config. Add aliases, environment variables, PATH additions.
~/.ssh/SSH keys and config for outbound connections (e.g. GitHub).
~/.local/User-local installs (pip --user, cargo, etc.) land here.
GitHub SSH: You can add a separate SSH key to ~/.ssh/ for GitHub access, letting you clone private repos and push from tharavad. Generate one with ssh-keygen -t ed25519 -C "tharavad" and add the public key to your GitHub account.

Installing packages

You don't have sudo access, but you can install packages into your home directory:

# Python packages (user install)
pip install --user flask

# Node packages (project-local, recommended)
npm install express

# Node packages (global, goes to ~/.local)
npm install -g some-tool

# Rust crates
cargo install some-tool
