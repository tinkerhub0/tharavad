SSH keys

tharavad uses SSH key authentication instead of passwords. You generate a keypair on your machine, then give the server your public key. The private key never leaves your computer.

What is an SSH key?

An SSH key is a pair of files — a private key and a public key — that together prove your identity to a remote server without sending a password over the network.

The private key (stored only on your machine) is like a physical key. The public key (given to the server) is like the lock. The server checks that your key fits the lock. You never send the private key anywhere.

Info: tharavad uses ed25519 keys, which are shorter, faster, and more secure than the older RSA format.

Step 1 — Generate a keypair

Run this in your terminal (on your own machine, not on tharavad):

ssh-keygen -t ed25519 -C "you@email.com"

Breaking down each part:

ssh-keygen
The key generation tool. Comes pre-installed on macOS, Linux, and Windows (since 2018).
-t ed25519
Sets the key type to ed25519 — a modern elliptic curve algorithm. Preferred over the older rsa type.
-C "you@email.com"
Adds a comment (the -C flag) to label the key — usually your email. This is just a human-readable tag; it doesn't affect authentication.

You'll be prompted to choose a save location (press Enter to accept the default ~/.ssh/id_ed25519) and optionally set a passphrase. A passphrase adds a second layer of protection — if someone gets your private key file, they still can't use it without the passphrase.

After running the command, two files are created:

~/.ssh/id_ed25519
Your private key. Never share this. Never upload it. Keep it only on your machine.
~/.ssh/id_ed25519.pub
Your public key. This is what you give to servers. It's safe to share.

Step 2 — Copy your public key

cat ~/.ssh/id_ed25519.pub
cat
Prints the contents of a file to your terminal. Short for concatenate.
~/.ssh/id_ed25519.pub
The path to your public key. ~ is shorthand for your home directory.

The output will look something like:

ssh-ed25519 AAAAC3NzaC1lZDI1NTE5AAAAI... you@email.com

Select and copy the entire line — from ssh-ed25519 to your email at the end.

Step 3 — Add it to your account

Paste your public key at:

https://auth.tharavad.xyz/if/flow/add-ssh-key/

This adds your public key to Authentik (the tharavad identity system). The server will then recognise your machine when you SSH in.

Already have a key? If you've used SSH before and have an existing ~/.ssh/id_ed25519.pub, you can skip the generate step and just add that key. You can also add multiple keys to your account — useful if you connect from more than one device.

Troubleshooting

Permission denied (publickey)

This usually means the server doesn't have your public key, or your account hasn't been approved yet. Double check that you pasted the .pub file (not the private key) and that an admin has approved your account.

ssh-keygen not found

On Windows, make sure you're using PowerShell or Windows Terminal (not old cmd). OpenSSH ships with Windows 10 and later. On older Windows, install Git for Windows which includes ssh-keygen.
