Hosting a project

Three things come together to host a project on tharavad: a reserved port, a running server inside tmux, and a public subdomain pointed at it by Caddy.

full flow
01
Reserve a port
Run tinker port to get an available port number assigned to your account. All users share the same network namespace, so ports are reserved to avoid conflicts.
Ports guide →
02
Start a tmux session
Processes started in a normal SSH shell die when you disconnect. tmux keeps your server running in the background persistently.
tmux guide →
03
Start your server on the reserved port
Run whatever server you want — Python's built-in HTTP server, a Node app, a Go binary — on the port number you reserved.
# example: static files
python3 -m http.server 45320

# example: node app
PORT=45320 node server.js
04
Expose a public subdomain
Use tinker caddy to route a subdomain to your port. This creates a live HTTPS URL at yourname.tharavad.xyz.
Subdomains guide →
Port lifetime: Reserved ports are automatically released after 7 days of inactivity. If your project stops working after a week, re-run tinker port and update your server and caddy config.
