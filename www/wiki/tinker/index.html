tinker CLI

Complete reference for all tinker commands. The tinker CLI manages port reservations and Caddy subdomain routing for your tharavad account.

Port management

tinker port
tinker port
Reserves a new available port for your account and prints the port number. The port is now exclusively yours — no other user can bind to it. Run this before starting any server you want to expose publicly.
tinker port
→ 45320
tinker port list
Lists all ports currently reserved under your account, along with when they were reserved and their last activity time. Useful when you have multiple projects and can't remember which port is which.
tinker port list
45320   reserved 3 days ago   last active: 2h ago
45321   reserved 1 day ago    last active: 10m ago
tinker port release <port>
Frees a reserved port so it's available to other users. Do this when you're done with a project. Does not stop any server running on the port — you need to kill that separately (e.g. inside tmux).
<port>The port number to release. Must be a port reserved by your account.
tinker port release 45320

Subdomain routing

tinker caddy
tinker caddy <subdomain> <port>
Creates a public HTTPS route at <subdomain>.tharavad.xyz that proxies to your server on <port>. The subdomain goes live immediately — no DNS propagation wait. TLS is handled automatically by Caddy. The port must already be reserved with tinker port and a server must be running on it.
<subdomain>The subdomain prefix. Must be unique across all users. Becomes subdomain.tharavad.xyz.
<port>The local port your server is listening on.
tinker caddy myapp 45320
→ live at https://myapp.tharavad.xyz

# multiple subdomains pointing to different ports
tinker caddy api    45320
tinker caddy blog   45321
tinker caddy list
Lists all active subdomain routes registered to your account. Shows the subdomain, the port it's routing to, and when the route was created.
tinker caddy list
myapp.tharavad.xyz  →  :45320   (created 2h ago)
blog.tharavad.xyz   →  :45321   (created 1d ago)
tinker caddy remove <subdomain>
Removes the public routing rule for a subdomain. The subdomain stops resolving immediately. Your server continues running on its port — it's just no longer publicly accessible. Use this to take a project offline without stopping the server.
<subdomain>The subdomain prefix to remove (not the full URL — just the part before .tharavad.xyz).
tinker caddy remove myapp

Quick reference

# ports
tinker port                        reserve a new port
tinker port list                   your reserved ports
tinker port release <port>         free a port

# subdomains
tinker caddy <subdomain> <port>    go live at subdomain.tharavad.xyz
tinker caddy list                  your active subdomains
tinker caddy remove <subdomain>    take it offline
