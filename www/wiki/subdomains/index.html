Subdomains

Once your server is running on a reserved port, tinker caddy wires up a public HTTPS URL at yourname.tharavad.xyz in seconds — no DNS config required.

How it works

tharavad runs Caddy — a web server that handles HTTPS certificates automatically — as its reverse proxy. When you run tinker caddy, it registers a routing rule in Caddy that forwards all traffic arriving at yoursubdomain.tharavad.xyz to your port on the local network. Caddy handles TLS so your URL is always HTTPS.

browser
→
yourname.tharavad.xyz (HTTPS)
→
Caddy (reverse proxy)
→
your server :45320

Exposing a subdomain

tinker caddy yourname 45320
tinker caddy
The subcommand that manages Caddy routing rules for your account.
yourname
The subdomain prefix. This becomes yourname.tharavad.xyz. Can be anything — your name, a project name, etc. Must be unique across all users.
45320
The port your server is running on. Must be a port you've reserved with tinker port.

After running this, your project is live at https://yourname.tharavad.xyz immediately. No waiting for DNS propagation.

HTTPS is automatic. Caddy provisions and renews TLS certificates via Let's Encrypt. You don't need to do anything for SSL — it just works.

Multiple subdomains

You can run multiple projects, each with its own subdomain pointing to a different port:

tinker caddy blog   45320   # → blog.tharavad.xyz
tinker caddy api    45321   # → api.tharavad.xyz
tinker caddy dev    45322   # → dev.tharavad.xyz

Each subdomain needs its own reserved port and a running server on that port.

Listing active subdomains

tinker caddy list

Shows all subdomains currently routed to your ports.

Taking a subdomain offline

tinker caddy remove yourname

Removes the routing rule. The subdomain stops resolving immediately. Your server keeps running on the port — it's just no longer publicly accessible.

Naming rules

Subdomain names must be unique across all tharavad users — not just within your account. If jake.tharavad.xyz is already taken by someone else, you'll get an error. Choose something specific to your project.

Reserved subdomains: Some names are taken by the platform itself — auth, vault, headscale, and wiki are used by tharavad infrastructure. Don't try to register those.
