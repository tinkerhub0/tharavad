Connecting via SSH

Once your account is approved and your SSH key is added, you can connect to tharavad from any terminal. tharavad runs SSH on a non-standard port (2222 instead of the usual 22).

Basic connection

ssh -p 2222 username@tharavad.xyz
ssh
The SSH client. Opens a secure encrypted shell session to a remote server.
-p 2222
Specifies the port. SSH normally runs on port 22, but tharavad uses 2222. Without this flag the connection will fail.
username@tharavad.xyz
Your tharavad username, followed by @, followed by the server address. Replace username with the one you registered.

On first connect, you'll see a fingerprint prompt asking if you trust the server. Type yes and press Enter. SSH saves the fingerprint so it won't ask again.

SSH config shortcut (optional)

Typing -p 2222 username@tharavad.xyz every time gets old fast. You can add a shortcut to your SSH config file that lets you just type ssh tharavad.

Open or create ~/.ssh/config and add:

Host tharavad
    HostName  tharavad.xyz
    Port      2222
    User      username
Host tharavad
The alias you'll type. Can be anything — tharavad, th, whatever you want.
HostName
The actual server address to connect to.
Port
The port number. Same as -p 2222 but saved in config so you don't type it each time.
User
Your username. Same as the username@ part of the command.

After saving the config, connect with just:

ssh tharavad
Note: The ~/.ssh/config file may not exist yet. Create it with touch ~/.ssh/config or just open it in any text editor. Make sure its permissions are correct: chmod 600 ~/.ssh/config.

After connecting

You'll land in a bash shell inside your home directory (/home/username). From here you can create files, run code, and use the tinker CLI to host projects publicly.

Important: When you close your terminal, any processes you started (web servers, etc.) will stop unless you started them inside a tmux session. See the tmux guide before hosting anything.
